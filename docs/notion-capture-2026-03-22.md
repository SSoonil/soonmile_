# Soonmile 작업 정리 (2026-03-22)

## 요약
- 로그인/소셜 인증 흐름 정리 및 자체 로그인 제거 방향 반영
- 여행 생성/수정 시 사진 업로드 및 핀 자동 분류(메타데이터 기반) 개선
- 핀 병합 규칙(300m) 및 핀-사진 매칭 오류 수정
- Trip 상세 UX 개선: 핀/커버/지도 인포윈도우 이미지에서 공통 갤러리 모달 제공

## 백엔드 변경
- `TripService` 핀 클러스터 기준 거리 `300m` 적용
- EXIF 메타데이터(좌표/시간) 기반 자동 핀 배정 로직 개선
- 업로드 후 인접 핀 병합 로직 보정(과도 병합 이슈 수정)
- 핀별 사진 조회(`getPinPhotos`) 기준 정렬/응답 경로 점검
- Java 21 기준 빌드/테스트 환경 재정렬

## 프론트엔드 변경
- 로그인 화면 UI 정리(요청된 배너/문구/버튼 구조 변경)
- 여행 생성/수정 시 업로드 파일 전달 경로 안정화
- 핀 상세에서 사진이 1장만 보이던 이슈 수정
  - 기본 유사사진 묶음 강제 로직 제거
  - `photoId` 기반 매핑으로 핀-사진 일치성 확보
- 갤러리 모달 추가
  - 핀 사진 클릭 시 큰 화면 보기
  - 좌/우 슬라이드 이동, 하단 썸네일 점프
  - 커버 이미지 클릭 시 여행 전체 사진 갤러리
  - 지도 핀 인포윈도우 이미지 클릭 시 동일 갤러리 연동
- 모달 위치 관련 실험적 스크롤/동적 위치 변경은 사용자 요청으로 원복

## 장애/원인 정리
- `failed to fetch` / `ERR_CONNECTION_RESET`:
  - 프론트 업로드 경로/백엔드 실행 상태/자바 버전 불일치 복합 이슈
- 서버 재시작 후 데이터가 사라져 보이는 현상:
  - 프론트가 시드/상태 기반 렌더, 백엔드는 일부 메모리 상태 사용
  - 목록 조회 API/완전 영속화 미구현 영역 존재

## 현재 상태
- 프론트/백엔드 서버 모두 종료 완료
  - 대상 포트: `5173`, `8080` 리스닝 없음

## 후속 권장 작업
1. 여행 목록 조회 API + 프론트 초기 로딩 연동
2. 핀/사진 엔티티 완전 영속화(재시작 후 동일 상태 보장)
3. 갤러리 모달 접근성 보완(포커스 트랩, 좌/우 아이콘 UI 고도화)
