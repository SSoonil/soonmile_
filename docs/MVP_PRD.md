# Soonmile MVP PRD (v0.1)

작성일: 2026-03-16  
문서 목적: 1차 MVP 범위와 성공 기준을 명확히 정의해 프론트/백엔드 구현 기준으로 사용

## 1. 제품 한 줄 정의
여행 사진을 업로드하면 위치/시간 메타데이터와 AI를 활용해 지도 기반으로 자동 정리하고, 그룹 단위로 추억을 관리하는 웹 서비스.

## 2. 문제 정의
1. 여행 후 사진이 많아 수동 정리 비용이 큼
2. 여러 명이 찍은 사진이 흩어져 관리가 어려움
3. 위치/시간 기반 회상이 어렵고, 의미 있는 스토리 구성이 힘듦

## 3. MVP 목표
1. 사진 업로드 후 지도 핀 자동 생성까지의 시간을 10분 이내로 제공
2. EXIF 유무와 무관하게 사용자가 최종적으로 모든 사진을 핀에 배정 가능
3. 그룹 단위 협업 업로드/조회 가능
4. AI 백그라운드 분석 결과(유사 사진, 저품질 추천, 캡션/제목 초안) 확인 가능

## 4. 핵심 사용자
1. 그룹장(OWNER): 여행/그룹 관리자, 권한 및 정책 관리
2. 멤버(MEMBER): 초대 수락 후 사진 업로드 및 열람

## 5. MVP 범위 (In Scope)
1. 계정 기반 로그인 상태에서 그룹 생성
2. 그룹 초대 및 초대 수락
3. 그룹 생성 시 위치정보/사진 활용 동의, 초대 수락 시 동의 재확인
4. 여행(Trip) 생성
5. 다중 사진 업로드
6. EXIF 파싱(위치, 촬영시간)
7. 지도 핀 자동 생성
8. 핀 군집 기준은 거리 100m
9. 핀 클릭 시 해당 핀의 사진 목록 표시
10. 핀 시간순 연결(여행 동선)
11. AI 작업 실행 버튼
12. AI 백그라운드 작업 상태 조회
13. AI 결과 제공
14. 유사 사진 그룹
15. 저품질/실패 후보 분리
16. 핀별 감성 제목/캡션 초안
17. EXIF 없는 사진 자동 분류 시도
18. 분류 불가 사진 수동 보정 프로세스 제공
19. 권한 정책
20. OWNER: 초대/권한/삭제/수정 가능
21. MEMBER: 업로드/조회 중심, 삭제/핵심 수정 불가

## 6. MVP 범위 (Out of Scope)
1. 고급 편집 기능(사진 보정, 필터, 동영상 편집)
2. 대규모 CDN/저장 비용 최적화 전략 확정
3. 결제/구독
4. 외부 SNS 자동 발행

## 7. 주요 사용자 플로우
1. OWNER가 그룹 생성 시 동의 체크 후 그룹 생성
2. OWNER가 멤버 초대 링크 발급
3. MEMBER가 초대 링크 수락 시 동의 체크 후 참여
4. 참여자가 Trip 생성 후 사진 업로드
5. 시스템이 EXIF 기반으로 핀 생성 및 100m 기준 군집화
6. EXIF 없는 사진은 AI 유사도 기반으로 인접 핀 또는 그룹에 자동 편입 시도
7. 자동 편입 실패 사진은 `미분류` 큐로 이동
8. 사용자가 지도/목록 UI에서 미분류 사진을 수동 배정
9. 사용자가 AI 요청 실행
10. 백그라운드 작업 완료 후 결과(유사/저품질/캡션/제목) 확인

## 8. 기능 요구사항
1. FR-01 그룹/초대/멤버십 관리가 가능해야 한다.
2. FR-02 동의 이력(누가/언제/무엇에 동의)을 저장해야 한다.
3. FR-03 사진 업로드 후 메타데이터 파싱 상태를 확인할 수 있어야 한다.
4. FR-04 위치 정보가 있는 사진은 100m 기준으로 핀 군집화되어야 한다.
5. FR-05 각 핀은 대표 좌표와 대표 시간(최초 또는 중앙값 기준)을 가진다.
6. FR-06 핀 상세에서 해당 핀 소속 사진을 시간순으로 보여줘야 한다.
7. FR-07 EXIF 없는 사진은 AI 기반 자동 분류를 시도해야 한다.
8. FR-08 자동 분류 실패 사진은 반드시 사용자 수동 처리 대상에 노출되어야 한다.
9. FR-09 AI 작업은 비동기 백그라운드 큐로 실행되어야 한다.
10. FR-10 AI 결과는 재조회 가능해야 하며 원본 데이터와 분리 저장되어야 한다.
11. FR-11 MEMBER는 사진 삭제/핵심 메타 수정 권한이 없어야 한다.
12. FR-12 OWNER는 멤버 관리, 여행 삭제, AI 결과 확정 권한을 가진다.

## 9. 비기능 요구사항
1. 다중 업로드 300장 기준 업로드 요청 수락은 5초 내 시작
2. 지도 초기 렌더 시 핀 데이터 로딩 p95 2초 이내
3. AI 작업은 비동기 처리, UI는 폴링 또는 SSE로 상태 갱신
4. 모든 주요 변경 작업은 감사 로그 또는 이력 추적 가능해야 함
5. 위치/사진 관련 데이터는 인증 사용자만 접근 가능해야 함

## 10. 성공 지표 (MVP)
1. 업로드 사진 중 자동 분류 성공률 70% 이상
2. 미분류 사진 수동 보정 완료율 95% 이상
3. AI 결과 확인 완료율 60% 이상
4. 그룹 초대 수락률 50% 이상

## 11. 리스크 및 대응
1. EXIF 없는 사진 비율이 높을 수 있음
2. 대응: AI + 수동 보정 UI를 필수 플로우로 설계
3. 위치 오차로 핀 품질 저하 가능
4. 대응: 수동 핀 재배정 및 핀 병합/분리 기능(간단 버전) 제공
5. 권한 오남용 가능
6. 대응: OWNER 고정 정책과 동의 로그, 삭제/수정 감사 로그 추가

## 12. 출시 기준 (Definition of Done)
1. 그룹 생성/초대/수락/동의 플로우가 실제 동작
2. 사진 업로드부터 지도 핀 확인까지 end-to-end 동작
3. 100m 군집화 및 핀 상세 사진 조회 동작
4. EXIF 없는 사진의 자동/수동 보정 플로우 동작
5. AI 백그라운드 작업 요청/상태/결과 조회 동작
6. OWNER/MEMBER 권한 제어 동작
